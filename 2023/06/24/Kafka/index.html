安装Kafka集群

1.上传安装包至/export/server路径下，并解压：

cd /export/server/

tar -zxvf kafka_2.12-2.4.1.tgz

2.修改配置文件

（1）进入配置文件目录

cd /export/server/kafka_2.12-2.4.1/config

(2)编辑配置文件

vi server.properties

为依次增长的:0、1、2、3、4,集群中唯一 id –》从0开始，每台不能重复，第一块要改的

broker.id=0

—-Logbasic——

数据存储的目录，第二块要改的

log.dirs=/export/data/kafka-logs

—zookeeper—-

指定 zk 集群地址，第四块要改的

zookeeper.connect=node1:2181,node2:2181,node3:2181

3.将配置好的kafka分发至node1和node2：

scp -r /export/server/kafka_2.12-2.4.1 root@node2:/export/server/

scp -r /export/server/kafka_2.12-2.4.1 root@node3:/export/server/

4.配置环境变量

5.在node2.node3上修改server.propertie中的brokeer编号：

vim /export/server/kafka/config/server.propertie

broker.id=1

broker.id=2

(broker.id 不能重复)

6.创建kafka一键启停脚本：

!/bin/bash

if [ $# -eq 0 ]

then

echo “please input param:start stop”

else

if [ $1 = start ]

then

for i in {1..3}

do

echo “${1}ing node${i}”

ssh node${i} “source /etc/profile;/export/server/kafka/bin/kafka-server-start.sh -daemon /export/server/kafka/config/server.properties”

done

fi

if [ $1 = stop ]

then

for i in {1..3}

do

ssh node${i} “source /etc/profile;/export/server/kafka/bin/kafka-server-stop.sh”

done

fi

fi

Kafka命令行操作

（1）创建topic：创建一个topic（主题）。Kafka中所有的消息都是保存在主题中，要生产消息到Kafka，首先必须要有一个确定的主题。

基本方式：./kafka-topics.sh –create –topic tpc_1 –partitions 2 –replication-factor 2 –zookeeper node1:2181

–replication-factor 副本数量

–partitions 分区数量

–topic topic 名称

（2）删除topic：bin/kafka-topics.sh –delete –topic tpc_1 –zookeeper node1：2181

（异步线程去删除）删除 topic,需要一个参数处于启用状态: delete.topic.enable = true,否则删不掉

使用 kafka-topics.sh 脚本删除主题的行为本质上只是在 ZooKeeper 中的 /admin/delete_topics 路径下 建一个与待删除主题同名的节点,以标记该主题为待删除的状态。与创建主题相同的是,真正删除主题的动作也是由 Kafka 的控制器负责完成的。

（3）查看topic：

列出当前系统中的所有 topic ：

bin/kafka-topics.sh –zookeeper node1:2181,node2:2181,node3:2181 -list

查看 topic 详细信息：bin/kafka-topics.sh –describe –topic tpc_1 –zookeper node1:2181

（4）增加分区数：bin/kafka-topics.sh –alter –topic tpc_1 –partitions 3 –zookeeper node1:2181

（5）****动态配置topic参数：添加、修改配置参数(开启压缩发送传输种提高kafka消息吞吐量的有效办法(‘gzip’, ‘snappy’, ‘lz4’, ‘zstd’))

bin/kafka-configs.sh –zookeeper node1:2181 –entity-type topics –entity-name tpc_1 –alter –add-config compression.type=gzip

删除配置参数

bin/kafka-configs.sh –zookeeper node1:2181 –entity-type topics –entity-name tpc_1 –alter –delete-config compression.type

（6）生产消息到Kafka进行消费：使用Kafka内置的测试程序，生产一些消息到Kafka的tpc_1主题中

bin/kafka-console-producer.sh –broker-list node1:9092, node2:9092, node3:9092 –topic tpc_1

>lu han

>luo yun xi

>kafka

消费消息(从头开始)

bin/kafka-console-consumer.sh –bootstrap-server node1:9092, node2:9092, node1:9092 –topic tpc_1 –from-beginning

指定要消费的分区,和要消费的起始 offset

bin/kafka-console-consumer.sh –bootstrap-server node1:9092,node2:9092,node3:9092 –topic tcp_1 –offset 2 –partition 0

（8）Kafka的基准测试

基准测试是一种测量和评估软件性能指标的活动。我们可以通过基准测试，了解到软件、硬件的性能水平。主要测试负载的执行时间、传输速度、吞吐量、资源占用率等。

Kafka中提供了内置的性能测试工具：

生产者：测试生产每秒传输的数据量（多少条数据、多少M的数据）

消费者：测试消费每条拉取的数据量

以下进行测试：

kafka-producer-perf-test.sh –topic tpc_1 –num-records 100000 –record-size 1024 –throughput -1 –producer-props bootstrap.servers=node1:9092 acks=1

kafka-producer-perf-test.sh –topic tpc_3 –num-records 100000 –record-size 1024 –throughput -1 –producer-props bootstrap.servers=node1:9092 acks=1

kafka-producer-perf-test.sh –topic tpc_6 –num-records 100000 –record-size 1024 –throughput -1 –producer-props bootstrap.servers=node1:9092 acks=1
